## 커뮤니케이션
- 모르거나 애매한 것은 무조건 질문할 것
- 처음부터 상세하게 잡지 않고, 확장 가능한 구조 선호
- 논의를 충분히 한 후에 구현에 들어갈 것 (먼저 구현하고 피드백 받는 방식 X)

## 제품 설계
- "묻지 말고, 먼저 제안하라" — 정보 부족해도 가안 먼저 작성
- 사용자 대화 최소화 (온보딩 마찰 최소화)
- [추측] 항목은 반드시 웹 검색 근거 + PM 확인 필수
- 과할 수 있는 설계도 일단 적용 후 실사용에서 검증 → 불필요하면 그때 제거

## 기술적 선호
- Claude Code 공식 기능(skills, agents, hooks) 적극 활용
- 문서 간 명시적 참조 구조 선호 (출처 추적 가능하도록)
- skill 이름은 구체적으로 (행위 + 대상 조합)

## 네이밍
- 영문명은 직관적이고 일관된 용어 선호
- 무리하게 세련된 용어로 바꾸지 말 것 — 변경 시 PM 확인 필수
