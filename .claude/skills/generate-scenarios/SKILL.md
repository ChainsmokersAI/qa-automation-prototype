# /generate-scenarios — 테스트 시나리오 생성

## 목적
context/에 구조화된 챗봇 정보를 기반으로, 챗봇의 품질을 검증할 수 있는 테스트 시나리오를 체계적으로 생성한다.
하나의 시나리오에서 여러 TC가 파생되므로, 시나리오의 방향이 올바른 것이 가장 중요하다.

---

## Step 0. 사전 확인

- `memory/MEMORY.md`를 읽고 이전 세션 상태를 파악한다.
- `context/` 파일들을 읽고 대상 챗봇을 숙지한다. context/가 비어있으면 `/learn-context`를 먼저 실행하도록 안내한다.
- `outputs/`에 이미 생성된 시나리오가 있는지 확인한다. 있다면 신규 생성이 아닌 추가/갱신 모드로 진행한다.

---

## Step 1. 시나리오 생성 전략 수립

context/ 내용을 분석하여 시나리오 생성 방향을 결정한다. 에이전트가 자율적으로 판단하되, 아래 우선순위를 따른다:

### 우선순위

1. **Happy Path (최우선)**: Core Capabilities와 User Flows를 기반으로 정상 동작 시나리오를 먼저 생성한다. 챗봇이 "잘 작동하는지"를 먼저 검증하는 것이 기본이다.
2. **Edge Case**: Out-of-Scope 경계를 기반으로 챗봇이 범위 밖 요청을 적절히 처리하는지, 경계 상황에서 어떻게 동작하는지를 테스트하는 시나리오를 생성한다.
3. **중요도 조정**: Issues(이슈) 항목이 있으면, 관련된 시나리오의 중요도를 높인다. Issues는 시나리오를 직접 생성하기보다 기존 시나리오의 우선순위를 결정하는 요인으로 활용한다.

---

## Step 2. 시나리오 작성

아래 포맷에 따라 시나리오를 작성한다.

```markdown
## S-{번호}: {시나리오 제목}

### 설명
{어떤 상황을 테스트하는지 구체적으로 서술}

### 테스트 목적
- {이 시나리오로 검증하려는 것 1}
- {이 시나리오로 검증하려는 것 2}

### 관련 컨텍스트
- [{항목명}](../context/{파일명}.md#{섹션-앵커})
- [{항목명}](../context/{파일명}.md#{섹션-앵커})

### 사전 조건
- {시나리오가 성립하기 위한 전제 조건}

### 중요도
{상 / 중 / 하} — {판단 근거}
```

### 작성 규칙

- **관련 컨텍스트**: 반드시 context/ 파일의 **섹션 레벨**까지 링크한다 (파일 레벨 X). context/ 변경 시 영향받는 시나리오를 추적하기 위함이다.
- **중요도**: Issues에 관련된 시나리오는 중요도를 상향 조정하고, 판단 근거에 해당 이슈를 명시한다.
- **시나리오 ID**: S-001부터 순차 부여. 기존 시나리오가 있으면 이어서 번호 부여.

---

## Step 3. PM 확인

생성된 시나리오 전체를 한 번에 PM에게 제시한다.

- 시나리오별 제목 + 설명 + 중요도를 요약 리스트로 먼저 보여준다
- PM이 요청하면 개별 시나리오 상세를 펼쳐서 보여준다
- PM 피드백 반영: 시나리오 수정, 삭제, 추가, 중요도 조정

---

## Step 4. 시나리오 저장

PM 확인이 완료되면 `outputs/scenarios/`에 저장한다.

- 파일 구성: 시나리오 수에 따라 단일 파일 또는 분할 (에이전트 판단)
- `outputs/index.md`에 생성 이력 기록

---

## Step 5. 메모리 업데이트

- `memory/task-log.md`에 작업 기록 (생성된 시나리오 수, 커버리지 요약)
- `memory/lessons-learned.md`에 교훈 기록 (있을 경우)
- `memory/MEMORY.md`의 프로젝트 현황 갱신
