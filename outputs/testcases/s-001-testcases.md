## TC-001: 네트워크 장비 개념 질문 (브릿지)

### 시나리오
[S-001: 강의 내용 기반 개념 질문-답변](../scenarios/edutap-scenarios.md#s-001)

### 관련 컨텍스트
- [Core Capabilities #2 강의 데이터 기반 답변](../../context/core-capabilities.md)
- [Course Data CX251104_1_6](../../context/course-data.md)

### 대화 유형
싱글턴

### 대화 흐름

#### Turn 1
- **사용자**: 브릿지가 정확히 뭐하는 장비에요? 허브랑 뭐가 달라요?
- **기대 응답**:
  - 브릿지가 LAN과 LAN을 연결하거나 LAN 안의 컴퓨터 그룹을 연결하는 장비라는 설명
  - 허브와의 차이점 (허브는 단순 신호 중계, 브릿지는 MAC 주소 기반 필터링 등)

### 평가 기준
- [ ] 브릿지의 역할(LAN 간 연결, 컴퓨터 그룹 연결)을 강의 내용에 근거하여 설명했는가
- [ ] 허브와의 차이점을 포함하여 답변했는가
- [ ] 강의에서 다루지 않은 내용을 지어내지 않았는가

### 대화 흐름

- **사용자**: 머신러닝으로 데이터 분석하려면 어떤 순서로 진행해야 돼요?

- [ ] 데이터 확보의 중요성을 언급했는가
- [ ] 강의 내용(비즈니스 목표 기반 기획)에 근거한 답변인가

## TC-004: 파인튜닝 개념 질문 후 후속 질문 (멀티턴)

### 시나리오
[S-001: 강의 내용 기반 개념 질문-답변](../scenarios/edutap-scenarios.md#s-001)

### 관련 컨텍스트
- [Core Capabilities #2 강의 데이터 기반 답변](../../context/core-capabilities.md)
- [Course Data CX251104_1_4](../../context/course-data.md)

### 대화 유형
멀티턴

### 대화 흐름

#### Turn 1
- **사용자**: 파인튜닝이 뭐예요? 그냥 처음부터 학습시키는 거랑 뭐가 다른 거예요?
- **기대 응답**:
  - 파인튜닝의 정의 (사전학습된 모델을 특정 분야에 맞게 추가 학습)
  - 처음부터 학습(프리트레이닝)과의 차이점

#### Turn 2
- **사용자**: 그럼 KPoEM이라는 모델은 어떤 데이터로 파인튜닝한 거예요?
- **기대 응답**:
  - KPoEM 모델에 사용된 입문 데이터셋에 대한 설명
  - 한국학중앙연구원에서 구축한 데이터라는 점 언급

#### Turn 3
- **사용자**: 입문 데이터셋을 만드는 게 왜 어렵다고 했죠?
- **기대 응답**:
  - 강의에서 언급한 "많은 시간과 노력이 필요하다"는 내용 반영
  - 데이터셋 구축의 어려움에 대한 구체적 설명

### 평가 기준
- [ ] 파인튜닝의 정의와 프리트레이닝 차이를 정확히 설명했는가
- [ ] KPoEM 모델과 입문 데이터셋에 대해 강의 내용에 근거하여 답변했는가
- [ ] 3턴에 걸쳐 대화 맥락이 일관되게 유지되었는가
- [ ] 이전 턴의 답변을 참조한 후속 질문에 적절히 응답했는가
