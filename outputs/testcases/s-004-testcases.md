### 대화 유형

### 대화 흐름

#### Turn 1
- **사용자**: 강의에서 arrow function 예시로 나온 코드 다시 한번 보여줘.
- **기대 응답**:
  - 강의 자료 내 arrow function 예시 코드 블록 인용
  - JavaScript 코드 펜스(```` ```js ````)로 렌더링

#### Turn 2
- **사용자**: 이 코드에서 괄호 없는 버전이랑 있는 버전 차이가 뭐야?
- **기대 응답**:
  - 단일 파라미터 시 괄호 생략 가능, return 생략 가능 등 규칙 설명
  - 강의에 나온 규칙 인용

#### Turn 3
- **사용자**: `const square = x => x * x;` 여기에 파라미터 2개 추가하면 어떻게 써야 해?
- **기대 응답**:
  - 수정된 코드 예시 (`(x, y) => x * y` 등) 제시
  - 괄호 필수 규칙 설명

#### Turn 4
- **사용자**: 그럼 본문이 여러 줄이면 어떻게 써?
- **기대 응답**:
  - 중괄호 본문 + 명시적 return 설명
  - 코드 예시 포함

#### Turn 5
- **사용자**: 일반 function 키워드로 바꾸면 어떻게 돼? 동일한 기능으로.
- **기대 응답**:
  - function 선언형 변환 예시 제시
  - this 바인딩 차이 언급 (강의 기반)

### 평가 기준
- [ ] 코드 블록이 JS 언어 태그로 렌더링되는가
- [ ] 강의 내 코드 예시와 일치하는가 (임의 코드 생성 금지)
- [ ] 변환 코드가 문법적으로 올바른가
- [ ] this 바인딩 차이 언급이 강의 기반인가

## TC-011: KPoEM 파인튜닝 코드 질문

### 시나리오
[S-004: 코드 스니펫 이해·수정 질문](../scenarios/01-happy-path.md#s-004-코드-스니펫-이해수정-질문)

### 관련 컨텍스트
- [Flow 4. 코드/실습 질문](../../context/04-user-flows.md#flow-4-코드실습-질문-javascript--ai-파인튜닝-클립)

### 대화 유형
멀티턴

### 대화 흐름

#### Turn 1
- **사용자**: KPoEM 파인튜닝 강의에 나온 코드 중에 데이터셋 로딩 부분 설명해줘.
- **기대 응답**:
  - 강의 내 데이터셋 로딩 코드 인용 (있다면)
  - 없으면 "해당 코드 구간은 확인되지 않는다"는 정직한 응답

#### Turn 2
- **사용자**: 이 코드가 왜 이렇게 작성됐는지 한 줄 한 줄 설명해줘.
- **기대 응답**:
  - 코드 라인별 설명
  - 강의 기반 설명 + 라이브러리 동작 근거

#### Turn 3
- **사용자**: 데이터 전처리 단계에서 꼭 해야 할 건 뭐야?
- **기대 응답**:
  - 강의에서 강조한 전처리 단계 나열
  - 코드와 연결

#### Turn 4
- **사용자**: 내가 이 코드 따라하다가 `KeyError` 나왔어. 왜 그럴까?
- **기대 응답**:
  - 일반적인 KeyError 원인 (키 오타, 데이터 구조 불일치 등) 설명
  - "정확한 원인은 에러 스택을 보여주세요" 식의 추가 정보 요청
  - 강의 기반 문제 해결 접근법

#### Turn 5
- **사용자**: 스택 트레이스 알려줄게: `KeyError: 'label'`. 해결 방법은?
- **기대 응답**:
  - `label` 키 누락·오타 가능성 진단
  - 해결 절차 제시 (print로 key 확인, rename 등)
  - 강의 예시와의 차이 비교

### 평가 기준
- [ ] 강의에 없는 코드는 "확인되지 않는다"고 정직하게 답변하는가
- [ ] 디버깅 제안이 구체적이고 실행 가능한가
- [ ] 에러 해결 접근이 강의 맥락과 연결되는가

## TC-012: 수강생 오류 코드 교정 요청

### 시나리오
[S-004: 코드 스니펫 이해·수정 질문](../scenarios/01-happy-path.md#s-004-코드-스니펫-이해수정-질문)

### 관련 컨텍스트
- [Flow 4. 코드/실습 질문](../../context/04-user-flows.md#flow-4-코드실습-질문-javascript--ai-파인튜닝-클립)

### 대화 유형
멀티턴

### 대화 흐름

#### Turn 1
- **사용자**: 내가 쓴 코드인데 실행이 안돼. `const items = [1,2,3]; items.map((x) => { x * 2 });`
- **기대 응답**:
  - 중괄호 블록에서 return 누락 문제 진단
  - 수정 코드 제시: `items.map((x) => x * 2)` 또는 `items.map((x) => { return x * 2 })`

#### Turn 2
- **사용자**: 왜 그렇게 하면 되는데? 강의에서 이 부분 다뤘었나?
- **기대 응답**:
  - 화살표 함수 본문 규칙 (중괄호 → 명시적 return 필요) 설명
  - 강의 내 관련 구간 인용 or 안내

#### Turn 3
- **사용자**: 그럼 ES11에서 이거 더 편하게 쓰는 방법은 없어?
- **기대 응답**:
  - 강의 범위 내 ES11 기능 중 관련 있는 것 언급 or "해당 개선은 강의에 없어요" 정직 고지
  - 할루시네이션 금지

#### Turn 4
- **사용자**: map 말고 for문으로 바꾸면 어떻게 돼?
- **기대 응답**:
  - for 문 변환 예시 코드 제시
  - let vs var 스코프 차이 간단 언급

#### Turn 5
- **사용자**: 고마워. 근데 내가 원래 뭘 하려고 했던 건지 챗봇이 기억해?
- **기대 응답**:
  - Turn 1 맥락 (배열 각 원소 2배) 상기
  - 대화 흐름 유지 확인

### 평가 기준
- [ ] 오류 진단이 정확한가
- [ ] 수정 코드가 문법적으로 동작하는가
- [ ] 강의 외 지식을 덧붙일 때 "강의 외"임을 표시하는가
- [ ] Turn 5의 맥락 유지가 작동하는가
