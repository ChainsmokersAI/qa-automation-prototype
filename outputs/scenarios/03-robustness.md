# Test Scenarios — Robustness

Issues(이슈) 항목 기반의 고위험·고가치 시나리오. Happy Path와 겹치는 듯 보이지만, 초점은 "챗봇이 까다로운 조건에서 버티는가"에 있다.

---

## S-015: RAG 검색 실패 유발 (모호어·오타·동의어·줄임말)

### 설명
수강생이 모호한 용어, 오타, 동의어, 줄임말 등으로 질문하여 RAG 검색 품질의 하한을 확인한다 (예: "머신러닝 기획이 뭐였지?" vs "ML 기획 설명해줘" vs "데이터 분석 기획 개요"). 93.8% 검색 정확도의 나머지 6.2% 실패 영역을 타겟팅.

### 테스트 목적
- 동일 의도의 다양한 표현에 대해 동일한 클립·동일한 품질의 답변이 나오는가
- 오타 발생 시 의도를 추론하는가
- 검색 실패 시 "잘 못 찾았어요, 다시 말해주실래요?" 식으로 정직하게 실패를 드러내는가
- 잘못된 청크를 검색해 엉뚱한 답을 내지는 않는가

### 관련 컨텍스트
- [Issues #1 RAG 검색 실패](../../context/06-issues.md#1-rag-검색-실패)

### 사전 조건
- 동일 질문의 표현 변형 세트가 준비되어 있음 (TC 단계에서 정의)

### 중요도
**상** — Issues #1 직결. 검색 정확도는 RAG 챗봇의 품질 상한을 결정하는 가장 본질적 지표.

## S-016: 소크라테스식 응답 피로도 제어 (직답 vs 유도 판단)

### 설명
수강생이 단순 사실 질문(예: "ES6는 언제 발표됐어?", "OSI 7계층에서 네트워크 계층은 몇 층이야?")을 던졌을 때, 챗봇이 기계적으로 질문을 되돌리지 않고 직답을 주는지 검증한다. 반대로 개념 이해 질문에는 소크라테스식 유도가 작동해야 한다.

### 테스트 목적
- 사실 질문 vs 개념 이해 질문을 구분하는가
- 사실 질문에 직답을 주는가
- 개념 이해 질문에 소크라테스식 유도를 작동시키는가
- 소크라테스식 유도가 연속 3턴 이상 반복되지 않는가 (피로도 상한)

### 관련 컨텍스트
- [Issues #4 소크라테스식 응답 피로도](../../context/06-issues.md#4-소크라테스식-응답-피로도)

### 사전 조건
- 없음

### 중요도
**중** — 학습 경험의 핵심이지만, 기본 답변 품질(S-001)이 검증된 후에 의미 있음.

## S-017: 비정형 콘텐츠 이해 (표·다이어그램·수식)

### 설명
수강생이 강의 자료 내의 표(컴활 1급 함수표), 네트워크 다이어그램(정보처리기사 네트워크 장비), 수식(머신러닝 기획) 등 비정형 콘텐츠에 대해 질문한다. OCR·STT 기반 처리의 한계 영역.

### 테스트 목적
- PDF 내 표·그림·수식 영역을 RAG가 정확히 파싱하는가
- SRT 자막 기반 답변이 슬라이드 시각 정보와 일관되는가
- "강의 7분 30초에 나오는 다이어그램 설명해줘" 같은 시점 참조 질문에 대응하는가
- 처리 불가 시 정직하게 고지하는가 ("이 부분은 영상을 직접 확인해주세요" 등)

### 관련 컨텍스트
- [Issues #5 특수 콘텐츠 처리](../../context/06-issues.md#5-특수-콘텐츠-처리)

### 사전 조건
- 해당 클립에 표/다이어그램/수식이 실제로 존재하는 것을 PDF·SRT에서 확인

### 중요도
**중** — OCR·STT 한계는 RAG 챗봇의 구조적 취약점이지만, 사용자가 가장 답답해하는 실패 패턴 중 하나.

## S-018: 언어·톤 일관성 (반말·존대·한영 혼용)

### 설명
수강생이 반말로 질문하거나 존대로 질문하거나, 한국어와 영어를 혼용한 질문을 한다 (예: "ES6 arrow function 설명해줘", "What is backpropagation? 한국어로 설명해줘"). 챗봇의 톤이 일관되게 유지되고 언어 전환이 자연스러워야 한다.

### 테스트 목적
- 수강생 반말/존대에 대한 챗봇 톤이 일관된가 (PM 정책 확인 필요)
- 한영 혼용 질문에서 핵심 용어·의도를 정확히 파악하는가
- 답변 언어가 예측 가능한 규칙을 따르는가 (질문 언어 따라가기 vs 고정)
- 언어 전환 시 강의 자료 참조 품질이 유지되는가

### 관련 컨텍스트
- [Issues #7 언어·톤 일관성](../../context/06-issues.md#7-언어톤-일관성)

### 사전 조건
- 챗봇 톤 정책이 정의되어 있음 (PM에게 `/run-simulation` 전까지 확인 요청)

### 중요도
**중** — Happy Path 품질에 영향을 주지만, 설계 단계에서 명확한 정책 기준이 있어야 평가 가능.
