# Test Scenarios — Happy Path

에듀탭 챗봇이 정상적으로 동작하는 핵심 시나리오. Core Capabilities + User Flows 기반.

---

## S-001: 강의 시청 중 개념 질문 (핵심 Happy Path)

### 설명
수강생이 `CX251104` 과정의 특정 클립을 시청하던 중 개념/용어가 이해되지 않아 챗봇에게 질문하고, 챗봇이 해당 클립의 강의 자료(PDF+SRT)를 RAG로 참조하여 답변하는 기본 시나리오.

### 테스트 목적
- 현재 재생 중인 클립 맥락을 챗봇이 올바르게 인식하는가
- 답변이 해당 클립의 실제 강의 내용에 근거하는가 (할루시네이션 없음)
- 6개 클립 각각에 대해 기본 개념 질문이 정확하게 처리되는가

### 관련 컨텍스트
- [실시간 1:1 대화형 학습 지원](../../context/03-core-capabilities.md#실시간-11-대화형-학습-지원)
- [Flow 1. 강의 시청 중 개념 질문](../../context/04-user-flows.md#flow-1-강의-시청-중-개념-질문-핵심-플로우)
- [연동된 강의 (지식 베이스)](../../context/03-core-capabilities.md#본-qa-세션에-연동된-강의-지식-베이스)

### 사전 조건
- 챗봇에 CX251104 6개 클립의 PDF+SRT가 연동되어 있음
- 수강생이 특정 클립을 시청 중(또는 시청 직후) 상태

### 중요도
**상** — Core Capability의 가장 기본 동작이자 모든 플로우의 출발점. 여기가 실패하면 나머지 시나리오는 의미가 없음.

---

## S-002: 소크라테스식 심화 학습 유도

### 설명
수강생의 후속 질문에 대해 챗봇이 직답을 주는 대신 질문으로 되돌려 사고를 유도하고, 학습자의 답변을 통해 이해도를 점검한 뒤 심화·추가 설명으로 이어가는 시나리오.

### 테스트 목적
- "질문 → 심화 → 점검" 사이클이 자연스럽게 작동하는가
- 학습자 답변의 정오에 따라 적절한 방향으로 분기하는가
- 소크라테스식 유도가 학습자를 답답하게 하지 않는 수준에서 조절되는가

### 관련 컨텍스트
- [실시간 1:1 대화형 학습 지원](../../context/03-core-capabilities.md#실시간-11-대화형-학습-지원)
- [Flow 1. 강의 시청 중 개념 질문](../../context/04-user-flows.md#flow-1-강의-시청-중-개념-질문-핵심-플로우)

### 사전 조건
- S-001의 기본 동작이 정상
- 수강생이 개념 질문 후 "더 자세히", "예시 들어줘" 등 심화 의도를 표명

### 중요도
**중** — 에듀탭의 차별화 포인트지만, 기본 답변이 먼저 검증되어야 의미가 있음.

## S-003: 복습 퀴즈 자동 생성

### 설명
수강생이 시청한 클립에 대해 "퀴즈 내줘", "문제 풀어보고 싶어" 등으로 요청하면, 챗봇이 강의 내용 기반으로 퀴즈를 생성하고 수강생 답변에 대해 정답·해설을 제공한다. 오답 시 관련 강의 구간 안내까지 이어진다.

### 테스트 목적
- 퀴즈 내용이 실제 강의 범위를 벗어나지 않는가
- 퀴즈 난이도가 클립 레벨(`clip_level: 1`)에 적합한가
- 정답 해설이 강의 자료와 일치하는가
- 오답 시 피드백과 재학습 유도가 작동하는가

### 관련 컨텍스트
- [학습 콘텐츠 자동 생성](../../context/03-core-capabilities.md#학습-콘텐츠-자동-생성)
- [Flow 2. 복습 퀴즈 요청](../../context/04-user-flows.md#flow-2-복습-퀴즈-요청)

### 사전 조건
- 수강생이 특정 클립을 시청 완료
- 챗봇이 해당 클립의 PDF+SRT에 접근 가능

### 중요도
**상** — 에듀탭 공식 자료에 명시된 핵심 기능. 사용자가 자주 요청할 것으로 예상되는 고빈도 플로우.

## S-004: 코드 스니펫 이해·수정 질문

### 설명
수강생이 `CX251104_1_5` (JavaScript ES6/ES11) 또는 `CX251104_1_4` (KPoEM 파인튜닝) 클립을 시청하면서 코드 스니펫을 이해하거나 수정하기 위해 챗봇에게 질문하는 시나리오. 코드 블록을 포함한 응답 렌더링과 강의 내 예시 기반 설명이 핵심.

### 테스트 목적
- 코드 블록이 올바르게 렌더링되는가 (마크다운 코드 펜스, 언어 하이라이트)
- 강의 내 예시 코드와 일치하는 설명을 제공하는가
- 코드 수정 요청에 대해 강의 범위 내에서 정확한 수정안을 제시하는가
- 수강생이 제공한 코드에 오류가 있을 때 올바르게 진단하는가

### 관련 컨텍스트
- [Flow 4. 코드/실습 질문](../../context/04-user-flows.md#flow-4-코드실습-질문-javascript--ai-파인튜닝-클립)
- [Issues #5 특수 콘텐츠 처리](../../context/06-issues.md#5-특수-콘텐츠-처리)

### 사전 조건
- 수강생이 JS 또는 KPoEM 클립 시청 중/후
- 챗봇이 해당 클립 PDF 내 코드 예시에 접근 가능

### 중요도
**중** — 코드 학습 클립에 국한되지만, 해당 클립 수강생에게는 핵심 경험. 렌더링 결함 시 체감 품질이 크게 저하.

---

## S-005: 자격증 시험 대비 요약·예상 문제 요청

### 설명
수강생이 `CX251104_1_3` (컴활 1급) 또는 `CX251104_1_6` (정보처리기사 네트워크) 클립을 시청하면서 "시험에 어떻게 나오나요?", "핵심만 요약해줘", "예상 문제 내줘" 같은 시험 대비 요청을 하는 시나리오.

### 테스트 목적
- 강의 범위 내에서 정확한 요약을 제공하는가
- 예상 문제가 실제 시험 유형에 부합하면서 강의 범위를 벗어나지 않는가
- "합격 보증" 같은 단정적 표현을 피하는가 (Out-of-Scope 규칙 준수)

### 관련 컨텍스트
- [Flow 5. 자격증/시험 대비 질문](../../context/04-user-flows.md#flow-5-자격증시험-대비-질문-컴활정보처리기사-클립)
- [Out-of-Scope: 성적·합격 보증](../../context/05-out-of-scope.md#교육-범위-밖)

### 사전 조건
- 수강생이 컴활 또는 정보처리기사 클립 시청 중/후

### 중요도
**중** — 자격증 학습자는 요약·예상 문제를 자주 요청. 동시에 "합격 보증" 오버클레임 리스크 상존.

## S-006: 여러 클립 교차 참조 질문

### 설명
수강생이 여러 클립에 걸친 주제(예: "빅데이터 분석 기획"과 "KPoEM 파인튜닝"의 관계, "정보처리기사 네트워크 장비"와 "직업윤리"의 접점 등)를 질문하면, 챗봇이 두 클립의 내용을 RAG로 조회해 비교·연결한다.

### 테스트 목적
- 여러 클립에서 관련 정보를 동시에 검색·종합하는가
- 클립 간 우선순위·출처를 명시하는가
- 클립 간 상충 정보가 있을 때(예: ES6 vs legacy 문법) 올바르게 처리하는가

### 관련 컨텍스트
- [Flow 3. 여러 클립 교차 참조](../../context/04-user-flows.md#flow-3-여러-클립-교차-참조)
- [Issues #3 멀티 클립 교차 질의](../../context/06-issues.md#3-멀티-클립-교차-질의)

### 사전 조건
- 수강생이 여러 클립을 시청한 상태

### 중요도
**중** — Issues #3에 해당하는 리스크 영역이지만, 단일 클립 질문이 먼저 정상화되어야 의미 있음.

---

## S-007: 학습 진도·이해도 점검

### 설명
수강생이 "내가 지금까지 뭘 배웠지?", "이해도 점검해줘" 같은 메타 질문을 하면 챗봇이 학습 현황 데이터 기반으로 요약·점검한다.

### 테스트 목적
- 학습 진도 데이터를 실제로 참조하는가 (대시보드 연동 여부)
- 수강한 클립 범위를 정확하게 반영하는가
- 이해도 점검 시 소크라테스식 질문으로 연결되는가

### 관련 컨텍스트
- [학습 현황 분석](../../context/03-core-capabilities.md#학습-현황-분석)
- [Flow 6. 학습 진도 점검](../../context/04-user-flows.md#flow-6-학습-진도-점검)

### 사전 조건
- 수강생 학습 진도 데이터가 챗봇에 노출됨 (API Spec 확인 후 검증)

### 중요도
**하** — 대시보드 연동에 의존하며 API Spec 확인 후에야 실제 테스트 가능. 현 단계에서는 인터페이스 수준만 검증.
